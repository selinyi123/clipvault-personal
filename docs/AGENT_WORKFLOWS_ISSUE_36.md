# Agent Workflow — Issue #36 v1.6.0 Release Gate

> Purpose: coordinate multiple agents without letting any agent falsely claim a release-gate item that requires Owner-controlled secrets, signed artifacts, physical-device QA, or final publication.

## 1. Operating rules

1. Do not expand product scope while closing Issue #36.
2. Do not change version metadata unless explicitly required by the release workflow.
3. Do not close Issue #36 from a commit message or PR body.
4. Do not use GitHub auto-close keywords near `#36`.
5. Do not claim a signed release from unsigned dry-run artifacts.
6. Do not claim manual QA from static analysis.
7. Do not claim artifact integrity from a green workflow run alone; verify downloaded bytes.
8. Do not publish `v1.6.0` until Owner approves publication.
