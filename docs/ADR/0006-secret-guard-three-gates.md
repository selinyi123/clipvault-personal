# ADR-0006: Secret Guard 三道闸门

状态：Accepted（2026-06-12）。**对原方案的强化**：原方案只在分类阶段检测一次。

## 决策
密钥拦截在三个独立位置实施（详见 THREAT_MODEL §4）：
- **闸门 A（捕获）**：两端 ingest 管线内、落库前判定，决定 FTS/派生是否发生。
- **闸门 B（出口）**：sync outbox 入队、Obsidian 写入、backup_queue 入队三处统一拒绝 is_secret=1。
- **闸门 C（备份序列化）**：backup worker 写 JSONL 前用当时最新规则复扫正文。

新增不变量（原方案缺失）：**密钥不进 FTS5 索引、不进日志、预览脱敏**——否则"不入库"形同虚设。

## 理由
单点检测的任何 bug（规则遗漏、调用顺序错、新增代码路径绕过）都会造成不可逆泄漏。三道闸门相互独立，规则升级后闸门 C 还能拦截历史漏网。

## 后果
- 同一内容最多被扫三次：自用数据量下性能可忽略。
- false positive 用 suspect 级别 + 显式释放流程（留审计）解决，不以降低灵敏度为代价。
