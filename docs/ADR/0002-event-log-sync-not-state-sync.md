# ADR-0002: 事件日志式同步，不做状态同步

状态：Accepted（2026-06-12）

## 背景
原方案只写了"WebSocket 实时同步 + 离线队列 + 去重"，没有定义语义。状态同步（比对两端全量状态）需要向量时钟/CRDT/墙钟信任，复杂且易错。

## 决策
同步 = 复制每台设备的追加型事件日志（outbox，设备内单调 seq）。
- clip 内容是不可变事实：`clip_new` 按 `(origin_device, seq)` + `content_hash` 双重幂等。
- 可变的只有标志位（pin/favorite/delete）：`clip_meta` 事件，字段级 LWW，平手 delete 赢。
- 游标续传：对端记录"我应用到你的哪个 seq"，重连从游标拉。

## 理由
- clip 天然 append-only，避开分布式状态合并的全部难题。
- 幂等让"重发、重连、双向同时推"都安全。
- 冲突域被压缩到三个布尔标志，LWW 的最坏后果是某次 pin 丢了——自用可接受。

## 后果
- 删除是软删（事件），不回收存储；自用数据量下不是问题。
- 多 Android 设备时桌面 hub 需转发事件（协议已支持 origin_device，实现推迟到真有第二台设备）。
