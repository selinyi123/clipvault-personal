# ADR-0007: v1 推荐引擎纯确定性（无 ML、无云）

状态：Accepted（2026-06-12）

## 决策
v1 的续词/推荐 = 前缀/子串匹配 + 频率（对数）× 指数时间衰减 + pinned/app 加权（SUG-1），SQL 预筛 + 内存重排。无任何模型、无任何网络调用。IME 端只查本地 Room 缓存。

## 理由
- 微信输入法的"舒适感"核心是**可预期**：常用的永远在老位置。确定性算法天然可预期、可解释、可单测。
- IME 按键延迟预算 30ms，本地 SQL 轻松达标，任何网络方案都达不到。
- 自用语料规模（数千条 memory + 数万 clip）用不到向量检索。

## 后果
- 不会有"语义联想"级别的惊喜；豆包式 AI 辅助归入 P2，且只能用户显式触发（点按钮），永不在按键路径上。
- 权重全部进 config，调感觉不用改代码。
